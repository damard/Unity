Game Mappings


Typically, each simulation system has its own Entity Type representation; Game Mappings determine how different simulations or game engines translate Entity Types. This section is used to create or modify mappings of Entity Types for incoming and outgoing Game Types.

LVC Game uses an Enumeration scheme to designate Entity Types. LVC Enumerations are made of seven space-separated integers that correspond to a DIS style entity type enumeration: 

	<v1> = Kind
	<v2> = Domain
	<v3> = Country
	<v4> = Category
	<v5> = Sub Category
	<v6> = Specific
	<v7> = Extra

To view all Game Entity Types that may be used for LVC mapping, click on All Game Types. This view can also be used to refresh the internal LVC Game database with the latest Game Entities by dynamically querying the game for all of its available entities.

The Incoming and Outgoing sections are used to modify or create new mappings. For example, all simulation data with the LVC Type of "1 1 0 1 1 2 0" can be mapped to an "m1abrams" on the incoming side, conversely you can chose to represent an "m1abrams" as "1 1 0 1 1 2 0" or another LVC Type on the outgoing side.

The Overrides section can be used to change the Unit Force on specific Game Types, or add a position offset to their Centre Of Mass.

Note: Entity Mappings can be dynamically added, removed and edited at runtime. For example if you notice that a mapping is missing by inspecting the Logs Summary or the Entity States table, you can add any missing mapping(s) dynamically and the corresponding entities or ammunition will become available in the simulation. However changes are only applied to newly created entities when editing existing mappings.
